### 前言
作为一名渴望成为黑客的前端仔😄，拓展自己的全平台能力尤为重要，而对于前端来说，**nodejs** 理应是最容易上手的选择，但是作为有一丢丢追求的我来说，我还是向往一个极致性能，可以无运行环境限制的语言来拓展我的能力，**无GC**，**内存安全**

### 什么是 Rust

> Rust 是由 Mozilla 团队打造的系统级编程语言 ，2010年首次亮相，2015年发布1.0稳定版。它的设计哲学围绕三个核心原则：性能，可靠性，生产力

**性能**-媲美 `C/C++` 的运行时效率
1. 零成本抽象：高级特性不引入运行时开销
2. 无垃圾回收：手动控制内存但无需手动释放
3. 极小运行时：适合嵌入式和性能敏感场景

**可靠性**-编译时消除内存错误
1. 所有权系统：根治空指针/野指针问题
2. 借用检查器：防止数据竞争和并发错误
3. 强类型系统：在编译期捕获逻辑错误

**生产力**-现代化开发体验
1. 内置包管理器：`cargo` (无比强大👍，尤其对比npm深有体会)
2. 丰富的错误提示和文档工具
3. 跨平台支持和WASM支持


### 为什么是 Rust

解决传统系统语言的痛点问题

- [x] 内存安全问题 - 段错误/内存泄露等 --- 所有权系统编译时拦截
- [x] 并发安全问题 - 数据竞争难以调试 --- 借用检查器防止竞争条件
- [x] 开发效率问题 - 依赖管理困难 --- `cargo` 统一构建/测试/发布


应用场景优势

- [x] 操作系统
- [x] 浏览器引擎
- [x] 虚拟机/容器运行时
- [x] 高性能网络服务 --- 例如：discord的后端服务；cloudflare的代理服务器边缘计算
- [x] WebAssembly领域
- [x] 嵌入式与物联网
- [x] 开发工具链


### 学习感受
最后说说学习rust的体会，相较于前端的 `javaScript` 天马行空的编码方式以及 `npm` 为人诟病糟糕的包管理机制以及那一大坨的 `node_modules`，初次上手rust让我印象最深刻的就是他的官方包管理工具`cargo`，让我感受到包管理器也能如此优雅 😢
